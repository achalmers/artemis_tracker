1. Executive Summary

ARTEMIS TRACKER is a standalone, production-ready HTML5 Single-Page Application (SPA) that visualizes the full 10-day trajectory of NASA's Artemis 2 crewed lunar flyby mission. The application displays an interactive 3D scene containing the Earth, Moon, and Orion spacecraft, together with the complete mission track from launch to splashdown.

✅ All core requirements met. The application is self-contained, runs entirely in a modern web browser, requires no server-side code, and includes a comprehensive automated test suite covering 40+ test cases across trajectory math, routing, and utility functions.

2.3 Hash-based App Router

The Router listens to window.hashchange events and maps hash fragments to view IDs: 

routes = {
  '/'          → 'view-visualization'  (default 3D view)
  '/telemetry' → 'view-telemetry'
  '/about'     → 'view-about'
}

Each navigation hides all .view divs and shows only the target, achieving SPA behavior without a server or build process. The active .nav-link is updated in sync.

2.4 Animation Loop

The single requestAnimationFrame loop in main.js advances the mission clock, calls scene.update() (Three.js render), and calls ui.update() (DOM refresh). Three.js rendering is skipped when the visualization view is not active, improving performance. 

// Playback: playSpeed is mission-hours per real second
trajectory.currentMET += dt_seconds × playSpeed;

3. Trajectory Model

3.1 Coordinate System

All positions are expressed in the Earth-Centered Inertial (ECI) frame:

  • Origin: Earth's centre of mass
  • X axis: Moon's initial direction at launch (T+0)
  • Y axis: North (Earth rotation axis, "up" in the 3D scene)
  • Z axis: Completes the right-hand system (perpendicular, in orbital plane)
  • Units: kilometres (km)

Scene units for Three.js rendering: 1 unit = 6 371 km = 1 Earth radius.

3.2 Moon Orbit

The Moon is modelled as a circular orbit in the XZ plane with a small Y-component representing the ~5.1° ecliptic inclination:

omega  = 2π / 655.2  rad/h
angle  = omega × MET_hours
moonX  = 384 400 × cos(angle)           [km]
moonY  = 384 400 × sin(angle) × 0.089   [km, inclination]
moonZ  = 384 400 × sin(angle)           [km]

3.3 Spacecraft Trajectory

The spacecraft path is defined by 27 control keyframes (MET in hours, X/Y/Z in km) covering all mission phases. Between keyframes, positions are computed with a Catmull–Rom spline for smooth, differentiable motion:

PhaseDurationDescription
Ascent0 – 0.15 hSurface to LEO insertion, 9 minutes
Parking Orbit0.15 – 2.94 h2 circular orbits at 200 km, T = 1.47 h
TLI Burn2.94 – 3.1 hTrans-Lunar Injection, raises apogee to Moon distance
Translunar Coast3.1 – 73 h~70-hour coast toward Moon along Bezier arc
Lunar Approach73 – 75 hEntering lunar sphere of influence
Lunar Flyby75 – 78 hFree-return swingby, closest approach ≈ 8 340 km
Return Coast78 – 232 h~6-day return coast to Earth
Reentry232 – 239 hAtmospheric entry, drogue/main chute deployment
Splashdown239 – 240 hPacific Ocean recovery

3.4 Derived Quantities

  • Velocity: Finite-difference numerical derivative of position, converted km/h → km/s
  • Distance to Earth: |position| (vector magnitude from origin)
  • Distance to Moon: |position − moonPosition| at same MET
  • Altitude: Distance to Earth − 6 371 km

4.3 Camera & Controls

OrbitControls provides drag-to-orbit, scroll-to-zoom, right-drag-to-pan. Five named camera presets are provided: Default, Top-Down, Side View, Earth Close-up, and Follow Spacecraft. The "Follow" mode lerps the orbit target toward the spacecraft each frame.

6. Test Suite

6.1 Overview

The test suite lives in tests/index.html + tests/tests.js. It runs entirely in the browser (no Node.js, no build tool). Tests are pure ES modules that import Trajectory directly. The DOM-dependent router tests run against a hidden minimal app shell embedded in the test page.

To run: open tests/index.html in a browser and click Run All Tests.

7. How to Run

7.1 Prerequisites

  • A modern browser: Chrome 89+, Firefox 108+, Edge 89+, or Safari 16.4+
  • Internet access on first load (to fetch Three.js and Google Fonts from CDN)
Important: ES modules require a web server — opening index.html directly as a file:// URL will be blocked by CORS. Use one of the methods below.

7.2 Recommended: Python dev server (no install)

cd artemis_tracker
python -m http.server 8080
# then open: http://localhost:8080

7.3 Alternative: Node.js (npx)

cd artemis_tracker
npx serve .
# follow the URL printed by serve

7.4 Alternative: VS Code Live Server extension

Right-click index.html in the Explorer panel → Open with Live Server.

7.5 Running the Tests

  1. Start the dev server (any method above)
  2. Navigate to http://localhost:8080/tests/ (or /tests/index.html)
  3. Click the ▶ Run All Tests button
  4. Review the grouped results and the summary bar

8. Design Decisions & Limitations

8.1 Trajectory Accuracy

The trajectory uses a parametric spline model (Catmull–Rom through 27 keyframes) rather than numerically integrating the equations of motion. This approach:

  • Produces visually accurate, smooth trajectories without a physics engine
  • Is computationally trivial — no ODE solver needed in the browser
  • Correctly captures all mission phases and the free-return geometry
  • Is not suitable for precision orbital mechanics research — errors of order 1–5% exist in velocities and timing

8.2 Real-time vs. Simulated Data

The application uses a pre-modelled trajectory. It does not connect to NASA's telemetry APIs. Actual Artemis 2 mission data could be substituted by replacing the _buildKeyframes() function in trajectory.js with live data.

8.3 Textures

Earth and Moon textures are generated procedurally via Canvas API, making the application fully self-contained after Three.js loads. No external image files are required. The Earth texture includes simplified continent outlines, polar ice caps, and a cloud overlay.

8.4 Scale

The scene renders at true relative scale (1 Three.js unit = Earth radius = 6 371 km). Earth–Moon distance is ~60 units in scene space. The spacecraft is intentionally rendered at a visible size (0.12 units = ~765 km equivalent) for usability.
